# BringAI — демо-чатбот для сайта

Облегчённая версия бота «Мо» для встраивания в сайт. Это **тот же мозг Rasa**
(NLU + форма записи + RAG-ответы + гардрейлы), но **без интеграций** WhatsApp / Instagram /
Telegram. Наружу смотрит только тонкий веб-адаптер, который зовёт ваш фронтенд.

## Как это устроено

```
Ваш сайт (браузер)
      │  POST /api/chat  {message, session_id}
      ▼
web_connector.py  (FastAPI, порт 8000)  ← единственный публичный вход
      │  POST /webhooks/rest/webhook  {sender, message}
      ▼
Rasa Core (5005, приватный) ──► Rasa Actions (5055, приватный)
                                   local-ответы → FAQ → RAG (OpenAI)
```

**Важно (безопасность):** порты 5005 и 5055 нельзя выставлять в интернет — только
`web_connector` (8000) за вашим reverse-proxy/TLS. Иначе кто угодно сможет слать боту
сообщения и жечь бюджет OpenAI.

## Что нужно фронтенду — контракт API

**`POST /api/chat`**
```jsonc
// запрос
{ "message": "сколько стоит?", "session_id": "abc123" }   // session_id опционален
// ответ
{ "session_id": "abc123", "replies": ["ответ 1", "ответ 2"] }
```
- `session_id` хранит контекст диалога (форма записи, история для RAG). Возьмите его из
  **первого** ответа и присылайте в каждом следующем запросе этого пользователя.
  Если не прислать — бот сгенерирует новый id на каждый запрос и будет «забывать» контекст.
- `replies` — список строк; показывайте их как отдельные сообщения бота.

Также есть `GET /health` (мониторинг) и `GET /` (простая тестовая страница чата —
только для локальной проверки; на проде фронтенд обслуживает ваш сайт).

## Запуск через Docker (рекомендуется)

```bash
cp .env.example .env      # впишите OPENAI_API_KEY (по желанию — см. ниже)
docker compose up -d      # поднимет rasa-core + rasa-actions + web-connector
```
Демо будет на `http://<сервер>:8000`. Проверка: откройте `http://localhost:8000/`.

## Запуск без Docker (локально, 3 процесса)

Нужен Python 3.10 и установленный Rasa 3.x:
```bash
python -m pip install "rasa>=3.6,<4.0" "rasa-sdk>=3.6,<4.0"
python -m pip install -r requirements.txt
cp .env.example .env

# в трёх терминалах:
rasa run actions                              # сервер действий :5055
rasa run --enable-api --cors "*"              # мозг Rasa :5005
uvicorn web_connector:app --port 8000         # веб-адаптер :8000
```

## OpenAI-ключ

RAG (ответы на редкие вопросы — миссия, KPI, устав и т.п.) требует `OPENAI_API_KEY` в `.env`.
**Без ключа бот тоже работает:** базовые вопросы (цена, расписание, пробное, адрес, скидки,
безопасность, возраст) отвечаются локально без OpenAI, а на остальное честно предлагает
уточнить у менеджера. `OPENAI_DAILY_CAP` ограничивает число RAG-ответов в сутки (защита бюджета).

## Что убрано по сравнению с боевой версией

- Мосты каналов: `green_bridge.py`, `telegram_bridge.py`, `wazzup_bridge.py` и их клиенты/сетапы.
- Уведомления менеджеру через WhatsApp/Green API. В демо `notify_manager` просто пишет в лог;
  заявка всё равно сохраняется в `data/leads.jsonl`.
- Зависимости `requests` и `python-docx` (нужны были только интеграциям).

## Если правите базу знаний

После изменения `data/qadam_knowledge.txt` пересоберите индекс эмбеддингов:
```bash
python rag_index.py
```
После изменения интентов/правил (`data/nlu.yml`, `data/rules.yml`) переобучите модель:
```bash
rasa train
```
